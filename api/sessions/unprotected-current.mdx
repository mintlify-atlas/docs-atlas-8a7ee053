---
title: Get unprotected current user
api: GET /api/sessions/unprotectedCurrent
description: Get the current user from unprotectedCookie JWT token
---

Retrieves the current user information from the `unprotectedCookie` JWT token set by the `/unprotectedLogin` endpoint.

<Warning>
  This endpoint reads from the `unprotectedCookie` which contains the full user object including sensitive data. Use the standard `/current` endpoint for production applications.
</Warning>

## Authentication

Requires the `unprotectedCookie` cookie to be present in the request. This cookie is set when you log in via `/api/sessions/unprotectedLogin`.

## Response

<ResponseField name="status" type="string">
  Success status ("success")
</ResponseField>

<ResponseField name="payload" type="object">
  Full user object from the JWT token
  
  <Expandable title="properties">
    <ResponseField name="_id" type="string">
      User's MongoDB ObjectId
    </ResponseField>
    
    <ResponseField name="first_name" type="string">
      User's first name
    </ResponseField>
    
    <ResponseField name="last_name" type="string">
      User's last name
    </ResponseField>
    
    <ResponseField name="email" type="string">
      User's email address
    </ResponseField>
    
    <ResponseField name="password" type="string">
      Hashed password (bcrypt)
    </ResponseField>
    
    <ResponseField name="role" type="string">
      User role (e.g., "user" or "admin")
    </ResponseField>
    
    <ResponseField name="pets" type="array">
      Array of pet IDs associated with the user
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

```bash cURL
curl -X GET http://localhost:8080/api/sessions/unprotectedCurrent \
  -b cookies.txt
```

## Response Example

```json
{
  "status": "success",
  "payload": {
    "_id": "507f1f77bcf86cd799439011",
    "first_name": "Jane",
    "last_name": "Doe",
    "email": "jane@example.com",
    "password": "$2a$10$abc123...",
    "role": "user",
    "pets": []
  }
}
```

## Differences from Standard Current

| Feature | `/current` | `/unprotectedCurrent` |
|---------|------------|----------------------|
| Cookie read | `coderCookie` | `unprotectedCookie` |
| Payload | User DTO only | Full user object |
| Includes password | No | Yes (hashed) |
| Production use | Recommended | Not recommended |

<Note>
  The standard `/current` endpoint returns only the DTO fields (name, role, email). This unprotected variant returns the complete database record including the password hash.
</Note>
