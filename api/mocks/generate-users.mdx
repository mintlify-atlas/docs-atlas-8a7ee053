## Overview

This endpoint generates 50 mock users with realistic data including names, emails, usernames, and securely hashed passwords. The users are not persisted to the database - they are returned as JSON for you to use as needed.

## Endpoint

```
GET /api/mocks/mockingusers
```

## Request

This endpoint requires no parameters or request body.

### Example Request

```bash cURL
curl -X GET http://localhost:8080/api/mocks/mockingusers
```

## Implementation Details

- **Fixed Quantity**: Always generates exactly 50 users
- **Password**: All users have the default password `coder123`, hashed with bcrypt (salt factor: 10)
- **Role Distribution**: Approximately 50/50 split between `user` and `admin` roles
- **Data Source**: Uses Faker.js `faker.name` and `faker.internet` modules
- **Not Persisted**: Users are generated in-memory and not saved to the database

- Populate UI components with realistic user data
- Create sample data for demonstrations
- Test user-related functionality with varied data
