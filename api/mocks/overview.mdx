## Overview

The Pet Adoption API provides mock data generation capabilities to help you quickly populate your database with realistic test data. These endpoints utilize [Faker.js](https://fakerjs.dev/) to generate random but realistic user and pet information.

## Use Cases

Mock data generation is useful for:

- **Development and Testing**: Quickly populate your database with test data during development
- **Demo Environments**: Create realistic demo data for presentations and product showcases
- **Performance Testing**: Generate large datasets to test application performance under load
- **UI/UX Testing**: Populate interfaces with varied data to test different scenarios

## Faker.js Integration

The API leverages Faker.js to generate:

- **User Data**: Names, emails, usernames, and roles
- **Pet Data**: Pet names, ages, breeds, and species
- **Realistic Variety**: Data includes natural variations and realistic combinations

## Data Characteristics

### Generated Users

- Random first and last names
- Valid email addresses
- Unique usernames
- Default password: `coder123` (hashed with bcrypt)
- Random role assignment (user or admin)
- Empty pets array by default

### Generated Pets

- Random pet names
- Ages between 1-15 years
- Random breeds
- Species: dog, cat, bird, or hamster

## Security Note

All generated users are created with the default password `coder123`, which is securely hashed using bcrypt with a salt factor of 10. Remember to change passwords in production environments.
