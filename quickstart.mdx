## Prerequisites

Before you begin, ensure you have the following installed:
- Node.js (v14 or higher)
- MongoDB (running locally or a connection URI)
- npm or yarn package manager

<Steps>
  <Step title="Clone and Install">
    First, clone the repository and install the dependencies:

    ```bash
    git clone <repository-url>
    cd pet-adoption-api
    npm install
    ```

    The project uses the following key dependencies:
    - **express** - Web framework
    - **mongoose** - MongoDB object modeling
    - **jsonwebtoken** - JWT authentication
    - **bcryptjs** - Password hashing
    - **cookie-parser** - Cookie handling
  </Step>

  <Step title="Configure Environment">
    Create a `.env` file in the root directory with your configuration:

    ```bash .env
    MONGO_URL=mongodb://localhost:27017/pet-adoption
    PORT=8080
    ```

    <Warning>
      Make sure your MongoDB instance is running before starting the server.
    </Warning>
  </Step>

  <Step title="Start the Server">
    Run the development server:

    <CodeGroup>
      ```bash npm
      npm run dev
      ```

      ```bash production
      npm start
      ```
    </CodeGroup>

    You should see:
    ```
    Listening on 8080
    ```

    The API will be available at `http://localhost:8080`
  </Step>

  <Step title="Make Your First API Call">
    Now let's register a new user, login, and fetch the user list.

    ### Register a New User

    ```bash curl
    curl -X POST http://localhost:8080/api/sessions/register \
      -H "Content-Type: application/json" \
      -d '{
        "first_name": "John",
        "last_name": "Doe",
        "email": "john.doe@example.com",
        "password": "securePassword123"
      }'
    ```

    <CodeGroup>
      ```javascript JavaScript
      const response = await fetch('http://localhost:8080/api/sessions/register', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({
          first_name: 'John',
          last_name: 'Doe',
          email: 'john.doe@example.com',
          password: 'securePassword123'
        })
      });

      const data = await response.json();
      console.log(data);
      // { status: "success", payload: "user_id_here" }
      ```

      ```python Python
      import requests

      response = requests.post(
          'http://localhost:8080/api/sessions/register',
          json={
              'first_name': 'John',
              'last_name': 'Doe',
              'email': 'john.doe@example.com',
              'password': 'securePassword123'
          }
      )

      print(response.json())
      # {'status': 'success', 'payload': 'user_id_here'}
      ```
    </CodeGroup>

    **Response:**
    ```json
    {
      "status": "success",
      "payload": "507f1f77bcf86cd799439011"
    }
    ```

    ### Login

    ```bash curl
    curl -X POST http://localhost:8080/api/sessions/login \
      -H "Content-Type: application/json" \
      -c cookies.txt \
      -d '{
        "email": "john.doe@example.com",
        "password": "securePassword123"
      }'
    ```

    <CodeGroup>
      ```javascript JavaScript
      const response = await fetch('http://localhost:8080/api/sessions/login', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
        },
        credentials: 'include', // Important for cookies
        body: JSON.stringify({
          email: 'john.doe@example.com',
          password: 'securePassword123'
        })
      });

      const data = await response.json();
      console.log(data);
      // { status: "success", message: "Logged in" }
      ```

      ```python Python
      import requests

      session = requests.Session()
      response = session.post(
          'http://localhost:8080/api/sessions/login',
          json={
              'email': 'john.doe@example.com',
              'password': 'securePassword123'
          }
      )

      print(response.json())
      # {'status': 'success', 'message': 'Logged in'}
      ```
    </CodeGroup>

    **Response:**
    ```json
    {
      "status": "success",
      "message": "Logged in"
    }
    ```

    <Note>
      The login endpoint sets a `coderCookie` cookie with your JWT token that expires in 1 hour.
    </Note>

    ### Get All Users

    ```bash curl
    curl -X GET http://localhost:8080/api/users \
      -b cookies.txt
    ```

    <CodeGroup>
      ```javascript JavaScript
      const response = await fetch('http://localhost:8080/api/users', {
        method: 'GET',
        credentials: 'include' // Include cookies
      });

      const data = await response.json();
      console.log(data);
      ```

      ```python Python
      # Using the same session from login
      response = session.get('http://localhost:8080/api/users')
      print(response.json())
      ```
    </CodeGroup>

    **Response:**
    ```json
    {
      "status": "success",
      "payload": [
        {
          "_id": "507f1f77bcf86cd799439011",
          "first_name": "John",
          "last_name": "Doe",
          "email": "john.doe@example.com",
          "role": "user",
          "pets": []
        }
      ]
    }
    ```
  </Step>
</Steps>

## API Base URL

By default, the API runs on:

```
http://localhost:8080
```

All endpoints are prefixed with `/api`, so the full base URL for API calls is:

```
http://localhost:8080/api
```

## Available Endpoints

The Pet Adoption API provides the following resource groups:

<CardGroup cols={2}>
  <Card title="Sessions" icon="key" color="#FF6B6B">
    Authentication endpoints for user registration and login
    - `/api/sessions/register`
    - `/api/sessions/login`
    - `/api/sessions/current`
  </Card>

  <Card title="Users" icon="users" color="#FF6B6B">
    User management endpoints
    - `/api/users`
    - `/api/users/:uid`
  </Card>

  <Card title="Pets" icon="paw" color="#FF6B6B">
    Pet management endpoints
    - `/api/pets`
    - `/api/pets/:pid`
  </Card>

  <Card title="Adoptions" icon="heart" color="#FF6B6B">
    Pet adoption management
    - `/api/adoptions`
    - `/api/adoptions/:aid`
  </Card>
</CardGroup>
